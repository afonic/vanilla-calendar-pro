# Read-Only Parameters

After initializing the calendar, additional read-only parameters are created.
These parameters contain information about the current state of the calendar, which is automatically updated when interacting with it.

<Info>
  The calendar's state changes with every interaction, which leads to the update of these parameter values. This is why they are available for reading only.
</Info>

## context.mainElement

`Type: HTMLElement`

```js
const calendar = new VanillaCalendar('#calendar');
console.log(calendar.context.mainElement);
```

The `mainElement` property contains the root HTML element in which the calendar was initialized.

- - -

## context.originalElement

`Type: HTMLElement`

```js
const calendar = new VanillaCalendar('#calendar');
console.log(calendar.context.originalElement);
```

The `originalElement` property contains the original root HTML element without any modifications, in which the calendar was initialized.

- - -

## context.inputElement

`Type: HTMLElement`

```js
const calendar = new VanillaCalendar('#calendar');
console.log(calendar.context.inputElement);
```

The `inputElement` property represents the root HTML element that is an input field.
